Navigation Overview

Three levels of navigation:

  • Sessions → groups of windows
  • Windows → tabs within a session
  • Panes → splits within a window

Navigation in tmux happens at three levels: sessions, windows, and panes. Understanding this hierarchy is essential for efficient tmux usage. Sessions contain windows, windows contain panes. You'll navigate between all three constantly.

KeyAction
j / k or arrowsNavigate
h / lCollapse / Expand
EnterSwitch to selected
xKill selected item
The session tree is one of the most powerful navigation features in tmux. It provides a visual overview of your entire tmux environment. Use vim-style navigation to move around, expand and collapse nodes, and quickly switch to any session, window, or pane.

Window Navigation

KeyAction
prefix + cCreate new window
prefix + nNext window
prefix + pPrevious window
prefix + 0-9Jump to window by number
prefix + wWindow list (interactive)
prefix + ,Rename current window
prefix + TabLast active window (oh-my-tmux)
Windows are like browser tabs. Create new ones with prefix-c, navigate sequentially with n/p, or jump directly by number. The window list (prefix-w) shows all windows across all sessions. Rename windows with prefix-comma to keep things organized.

Pane Navigation

KeyAction
prefix + h/j/k/lMove between panes (vim-style)
prefix + oCycle through panes
prefix + ;Toggle last active pane
prefix + qShow pane numbers, press number to jump
 
         k (up)
           ↑
           |
    h ← [pane] → l
           |
           ↓
       j (down)
Pane navigation uses vim-style h/j/k/l keys thanks to oh-my-tmux. This makes moving between panes feel natural if you're a vim user. The prefix-q command is useful for layouts with many panes - it briefly shows numbers on each pane that you can press to jump directly.

Mouse Navigation

KeyAction
prefix + m Toggle mouse mode

When Enabled

  • Click to select pane
  • Click window in status bar
  • Scroll to enter copy mode
  • Drag pane borders to resize

When Disabled

  • Terminal selection works normally
  • Better for copy/paste with terminal
Mouse mode is a personal preference. When enabled, it makes tmux more intuitive for mouse users but can interfere with terminal text selection. Toggle it with prefix-m to find what works best for your workflow. Many users keep it enabled and hold Shift to bypass tmux when selecting text.

Quick Returns

KeyAction
prefix + ; Last active pane
prefix + Tab Last active window

Workflow Example

Work in Pane A
→
Check Pane B
→
prefix + ;
→
Back to Pane A
These two keybindings are essential for efficient workflows. Instead of navigating through multiple panes or windows, you can quickly toggle back to where you were. This is especially useful when checking logs, running tests, or referencing documentation while coding.

Exercise 2 - Navigation

  1. Create 3 windows with prefix + c
  2. Navigate with prefix + n and prefix + p
  3. Use prefix + 1, prefix + 2 to jump directly
  4. Open session tree with prefix + s
  5. Navigate tree with j/k, expand with l
  6. Press q to exit tree
Bonus: Create panes and practice prefix + h/j/k/l
Walk through each step with the audience. Let them practice creating windows, navigating between them, and exploring the session tree. The session tree is particularly important to understand as it provides an overview of the entire tmux environment.
