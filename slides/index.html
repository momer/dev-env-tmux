Terminal multiplexer for your development workflow

Welcome to the Tmux Configuration Tutorial. This session will cover how to configure and customize tmux to enhance your development workflow. We'll explore keybindings, window management, and practical tips to make tmux work better for you.

What Is Tmux?

  • Terminal Multiplexer: Multiple sessions in one terminal window
  • Session Persistence: Sessions keep running after SSH disconnect or system reboot (with plugins)
  • Split Screen: Multiple panes in one view—divide your workspace instantly
  • Scriptable & Keyboard-Driven: Control everything from the keyboard, automate workflows

Quick Start

git clone [repo] ~/.config/dev-env-tmux
cd ~/.config/dev-env-tmux
make install        # Full setup (copies files)

# Alternatives:
make setup-symlink  # Use symlinks (for version control)
make setup-minimal  # Skip auto plugin install

Architecture

oh-my-tmux Base configuration framework with sane defaults
tpm Plugin manager (like vim-plug)
tmux-sensible Universal tmux settings
tmux-resurrect Save/restore sessions (pane contents, neovim)
tmux-continuum Auto-save every 15 min, auto-restore on start

File Locations

~/.tmux.conf → oh-my-tmux (do not edit)  |  ~/.tmux.conf.local → your customizations  |  ~/.tmux/plugins/ → installed plugins
The architecture is built on oh-my-tmux as a foundation. tpm handles plugin installation, just like vim-plug for neovim. tmux-sensible provides sensible defaults that most users want. tmux-resurrect and tmux-continuum work together for session persistence: resurrect saves the session state, continuum automates saving and restoring. The key distinction: never edit ~/.tmux.conf directly - all customizations go in ~/.tmux.conf.local

Key Bindings Preview

Key Action Notes
C-a or C-b Prefix Both work (oh-my-tmux)
prefix + | Vertical split Custom (intuitive)
prefix + - Horizontal split Custom
prefix + h/j/k/l Navigate panes vim-style (oh-my-tmux)
prefix + s Session tree Navigate all sessions
prefix + ? List all bindings Your reference
Pro Tip: Press prefix + ? to list all available keybindings whenever you forget a command!
Walk through each binding category: - Prefix: Both C-a and C-b work due to oh-my-tmux config. C-a is traditional screen style, C-b is default tmux. - Splits: The custom bindings using | and - are more intuitive than default % and " - Navigation: vim-style hjkl movement makes sense if you're a vim user. - Session tree: Quickly jump between sessions without typing session names. - Help: The ? binding is your reference guide during the presentation.

oh-my-tmux Extras

Key Action

prefix + r Reload configuration
prefix + m Toggle mouse mode
prefix + + Maximize pane to new window
Tip: These bindings are built into oh-my-tmux's base configuration and make tmux much more user-friendly right out of the box.
These additional keybindings come pre-configured with oh-my-tmux and provide quick access to common tasks. The edit local config binding is particularly useful for customization without touching the main configuration file. The reload binding lets you test changes immediately, and the mouse mode toggle is helpful for accessibility.
